    Design

       

    Aparência

    • Não dependa apenas de cor; ofereça outra pista (rótulo, padrão, ícone). WCAG 1.4.1 3, 1.4.3 4; NBR 5.11.1 1;
    • Textos ampliam ~200% sem quebrar layout nem ocultar conteúdo. WCAG 1.4.4 5, 1.4.10 6;
    • Disponibilize tema acessível (alto contraste / modo escuro). WCAG 1.4.3 4, 1.4.11 7; NBR 5.11.3; 5.11.2 1; Guia UK/BR 2;
    • Evite MAIÚSCULAS prolongadas, fontes muito condensadas e texto justificado. NBR 5.11 1; Guia UK/BR 2;
    • Nomes de controles descrevem a ação (não apenas “ícone de…”). WCAG 2.4.6 8, 4.1.2 9; NBR 5.13.7; 5.13.10 1; Guia UK/BR 2;
    • Parágrafos com ~80 caracteres por linha e fontes fluidas. NBR 5.12.6 1; Guia UK/BR 2;
    • Controles nomeados pelo propósito, não pela posição/visual. WCAG 4.1.2 9; NBR 3.42; 5.13.7; 5.13.10; 5.8.3 1; Guia UK/BR 2;
    • Ajuste de linhas ao viewport; cuidado com display:none/visibility:hidden. WCAG 1.3.1 10, 2.4.3 11, 4.1.2 9; NBR 1; Guia UK/BR 2;

    Animação

    • Nada pisca acima de 3 vezes/seg. WCAG 2.3.1 12, 2.3.2 13;
    • Todo movimento pode pausar/parar/ocultar. WCAG 2.2.2 14;
    • Respeite “reduzir movimento”. WCAG 2.3.3 15; NBR 1; Guia UK/BR 2;

    Contraste de cores

    • Texto com contraste adequado (normal e grande). WCAG 1.4.3 4; NBR 5.11.3; 5.11.2 1;
    • Ícones/bordas/indicadores com contraste mínimo. WCAG 1.4.11 7; NBR 5.11.4; 5.11.5 1;
    • Texto sobre imagem continua legível. WCAG 1.4.3 4, 1.4.11 7; NBR 1; Guia UK/BR 2;
    • Estados ativos/selecionados permanecem legíveis. WCAG 1.4.11 7; NBR 5.11.6 1;
    • Inclua seleção e bordas de inputs na checagem de contraste. WCAG 1.4.11 7; Guia UK/BR 2;

    Interação e foco

    • Foco visível e perceptível. WCAG 2.4.7 16; NBR 5.11.6; 5.10.5 1;
    • Foco não fica encoberto. WCAG 2.4.11 17, 2.4.12 18; NBR 5.10 1;
    • Alvos de toque com tamanho/afastamento confortáveis; alternativa ao arrastar. WCAG 2.5.8 19, 2.5.7 20;
