    Desenvolvimento Web

       

    Imagens

    • Imagens informativas: devem ter alt com descrição breve e significativa do conteúdo. WCAG 1.1.1 3; NBR 1;
    • Imagens decorativas: devem ser ignoradas por leitores de tela, utilizando alt="" (vazio) ou role="presentation". WCAG 1.1.1 3; NBR 1;
    • Imagens com legenda ou título: deve ter legenda ou título associado à imagem com texto claro e descritivo. WCAG 2.4.6 20; NBR 1;
    • Imagens em botões ou links o alt deve descrever a ação (ex: alt="Pesquisar"). WCAG 1.1.1 3; NBR 1;

    Vídeos e Mídia

    • Legenda sincronizada: todo vídeo com áudio deve possuir legendas. WCAG 1.2.2 5; NBR 1;
    • Transcrição textual: conteúdos em áudio devem ter transcrição equivalente. WCAG 1.2.1 4; NBR 1;
    • Audiodescrição: obrigatória quando informações visuais forem essenciais à compreensão. WCAG 1.2.5 6; NBR 1;
    • Controle de reprodução: não utilize autoplay, ofereça pausa, stop e volume ajustável. WCAG 1.4.2 11, 2.2.2 17; NBR 1;
    • Acessibilidade no player: garanta que todos os botões do player sejam acessíveis por teclado e tenham rótulos (aria-label). WCAG 2.1.1 14; Guia UK/BR 2;

    Navegação e Controles

    • Links e botões semânticos: links devem ter href válido e botões devem usar <button> com ação clara. WCAG 1.3.1 7, 4.1.2 32;
    • Atalho “Pular para o conteúdo”: inclua link no topo da página para ir direto à área principal. WCAG 2.4.1 18;
    • Aviso de nova aba ou janela: informe quando o clique abrir novo contexto. WCAG 3.2.2 25; Guia UK/BR 2;
    • Foco e ordem lógica: a navegação via teclado deve seguir ordem coerente. WCAG 2.4.3 19, 2.4.7 21;

    Formulários

    • Mensagens de erro claras: devem aparecer próximas ao campo e indicar como corrigir. WCAG 3.3.1 26, 3.3.3 28;
    • Autocompletar e dados persistentes: permita o uso de autocomplete e evite preenchimentos repetidos. WCAG 1.3.5 10, 3.3.7 29;
    • Sumário de validação: exibe lista de erros no topo após o envio. WCAG 3.3.1 26; Guia UK/BR 2;
    • Feedback visual e textual: deve fornecer feedback visual e textual em ações (envio, carregamento, sucesso, erro). NBR 1;
    • Campos obrigatórios: devem ser identificados textual e visualmente (ex: “*Campo obrigatório”). WCAG 3.3.2 27;
    • Evitar entrada redundante; autenticação acessível. WCAG 3.3.7 29, 3.3.8 30, 3.3.9 31; NBR 1;

    Estrutura e Semântica

    • Elementos nativos HTML: prefira <button>, <ul>, <table>, <form> em vez de divs genéricas. WCAG 1.3.1 7, 4.1.2 32;
    • Hierarquia de títulos: não pule níveis. (ex: <h1> direto para <h3>). WCAG 1.3.1 7; NBR 5.3.5 1;
    • Tabelas acessíveis: inclua <caption> e identifique cabeçalhos com <th scope=“col/row”>. WCAG 1.3.1 7; NBR 5.6.3, 5.6.5 1;
    • Ordem do DOM: deve seguir a ordem lógica de leitura e de tabulação, onde a navegação por teclado deve seguir a sequência visual. WCAG 1.3.2 8; NBR 5.13.6, 5.10.2 1;

    Modais e Diálogos

    • Foco no modal: o foco deve ir para o modal ao abrir e voltar ao elemento anterior ao fechar. WCAG 2.4.3 19, 2.4.7 21; NBR 5.1.4 1;
    • Fechamento Acessível (Sem Armadilhas): o usuário deve poder fechar o componente via teclado (tecla Esc ou Tab) e deve existir um botão de fechar visível. O foco não deve ficar preso. WCAG 2.1.2 15; NBR 5.1.6 1;
    • Evitar sobreposição: não utilize modais dentro de outros modais ou fullscreen forçado. Guia UK/BR 2;

    Dispositivos Móveis e Toque

    • Layout responsivo: deve ajustar-se sem rolagem horizontal e suportar zoom de até 200%. WCAG 1.4.10 13, 1.4.4 12;
    • Áreas de toque adequadas: botões e links devem ter tamanho e espaçamento suficientes para o toque. WCAG 2.5.8 24;
    • Orientação flexível: o conteúdo deve funcionar tanto em modo retrato quanto em paisagem. WCAG 1.3.4 9; NBR 1;
    • Alternativa ao arrastar; alvo mínimo. WCAG 2.5.7 23, 2.5.8 24; NBR 1;

    Teclado e Interação

    • Acessibilidade total via teclado: todas as funções devem ser operáveis sem mouse. WCAG 2.1.1 14; NBR 5.1.13 1;
    • Foco visível: o elemento ativo deve ter destaque claro e visível. WCAG 2.4.7 21, 2.4.11 22; NBR 5.1.1, 5.1.3 1;
    • Equivalentes ao mouseover: toda interação por mouse deve ter equivalente no teclado. WCAG 2.1.1 14; NBR 5.1.8, 5.1.9 1;
    • Atalhos de teclado: fornece atalhos quando apropriado e personalizáveis. WCAG 2.1.4 16; NBR 5.1.11 1; Guia UK/BR 2;

    • Testes automáticos: realizar testes automáticos com ferramentas como WAVE, Lighthouse, axe DevTools. NBR 1;
    • Testes manuais: teste com NVDA, VoiceOver e com usuários reais. NBR 1;
    • Documentação: registrar não conformidades e plano de correção. WCAG 5.1 33;
    • Registrar achados, priorizar correções e acompanhar.
    • Alvo: conformidade WCAG AA / NBR 17225. WCAG 2.2 (AA) 34; NBR 1;
